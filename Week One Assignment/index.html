Overview

HTML5 introduces meaningful structure to web documents. It promotes clarity for both human readers and search engines, laying the foundation for accessible and discoverable content.

Why Use Semantic HTML?

Improves Accessibility

Screen readers can better understand and navigate your content when it's structured using semantic elements like <header>, <main>, and <footer>.

Enhances SEO

Search engines rely on semantic HTML to understand page layout and relevance. Proper use of headings and sectioning helps with ranking.

Makes Code More Readable

Developers can quickly understand a page’s structure, making collaboration and maintenance easier.

Basic Page Structure

A well-structured HTML5 document includes elements such as:

  • <header> – Contains the site title or logo and navigation
  • <main> – The main content area of the page
  • <section> – Thematic grouping of content
  • <article> – Independent, reusable content blocks
  • <footer> – Copyright and contact info

Accessibility Considerations

Semantic HTML supports assistive technologies through landmarks and labels. Using aria-label and aria-labelledby helps define navigation and content regions more clearly.

SEO Best Practices

To improve search engine rankings:

  • Use one <h1> for the page title
  • Follow logical heading order: h2 for sections, h3 for subtopics
  • Include a descriptive <title> and <meta name="description">
  • Ensure all content is relevant and well-structured
